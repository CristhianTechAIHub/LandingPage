proyecto_01

Módulo de Industria 4.0

SENATI · Arquitectura Edge AI + Robótica Colaborativa

La IA no como módulo aislado, sino como el cerebro de la automatización. Celda industrial con arquitectura heterogénea donde una Jetson, una IPC y un PLC hablan el mismo idioma para guiar un cobot con precisión milimétrica — convirtiendo una "curiosidad de laboratorio" en una solución de ingeniería robusta.

  • Seguridad y Gateway · Edge AI NVIDIA Jetson Xavier NX como punto de entrada inteligente de la celda: autenticación biométrica por reconocimiento facial y gestión centralizada del flujo de comunicación con el PLC S7-1500 mediante protocolos TCP/IP industriales.
  • Visión para la Acción · Localización Pipeline de segmentación de instancias procesado en una IPC Industrial para detección y localización de piezas en escena. El núcleo fue la lógica de transformación de coordenadas: de píxeles del modelo a posiciones precisas en el espacio tridimensional, alcanzando precisión milimétrica para el movimiento del cobot.
  • Monitoreo en Tiempo Real · HMI Webserver embebido dedicado a la visualización en vivo de las inferencias del modelo de visión. Permite supervisión, diagnóstico y ajuste de parámetros desde cualquier dispositivo conectado a la red industrial, convirtiendo cada inferencia en información accionable para el operador.
  • Control de Calidad · Clasificación en el Borde Módulo TM NPU de Siemens integrado en el ciclo de control del PLC ejecutando ResNet-50 en tiempo real. La clasificación ocurre directamente en el controlador, eliminando latencias de comunicación externa y garantizando decisiones deterministas dentro del ciclo de PLC.

proyecto_02

Sistema de Sincronización de GMAO

Gestión de Mantenimiento · IA + Bases de Datos Heterogéneas

Plataforma de sincronización bidireccional entre dos sistemas de gestión de mantenimiento (GMAO), con detección automática de anomalías mediante un modelo Transformer entrenado sobre series temporales de sensores industriales. Las anomalías detectadas disparan automáticamente la creación de órdenes de trabajo en ambos sistemas.

  • Pipeline multi-base-de-datos unificado integrando Oracle, PostgreSQL, SQL Server e InfluxDB.
  • Modelo Transformer para detección de anomalías en series temporales de datos de sensores industriales.
  • Generación automática de órdenes de trabajo en el sistema maestro con reflejo instantáneo en el sistema esclavo.
  • Entrenamiento continuo del modelo con datos históricos almacenados en InfluxDB, sin intervención manual.
